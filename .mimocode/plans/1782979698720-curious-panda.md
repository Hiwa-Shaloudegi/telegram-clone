# PV Chat Creation — Implementation Plan

## Context

The Telegram clone app has chat messaging (send/edit/delete) already implemented. The missing piece is **creating a new private (PV) chat** when a user taps a contact in the contacts page. Currently, the `onTap` callback in `contacts_list.dart` is a no-op (commented out).

Additionally, there's a **bug**: the Dart `ChatType` enum uses `private` but the Supabase database uses `dm` as the `chat_type` value. `ChatModel.fromJson` does an exact match (`ChatType.values.firstWhere((e) => e.name == json['chat_type'])`) which would fail on `dm`.

## Changes

### 1. Fix chat_type mismatch (`ChatModel.fromJson`)

**File**: `lib/data/models/chat_model.dart` (line 50)

Change the `fromJson` factory to map `dm` → `ChatType.private`, matching how `ChatListItemModel.fromJson` already handles it:

```dart
chatType: ChatType.values.firstWhere(
  (e) => e.name == json['chat_type'],
  orElse: () => ChatType.private,
),
```

Also fix the `toJson()` method (line 63) to serialize `private` back to `dm`:

```dart
'chat_type': chatType == ChatType.private ? 'dm' : chatType.name,
```

### 2. Create Supabase RPC function `get_or_create_private_chat`

A PostgreSQL function that atomically finds an existing DM chat between two users, or creates one:

```sql
CREATE OR REPLACE FUNCTION get_or_create_private_chat(p_other_user_id UUID)
RETURNS UUID
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
  v_chat_id UUID;
  v_current_user_id UUID;
BEGIN
  v_current_user_id := auth.uid();

  -- Look for existing DM chat between these two users
  SELECT cm1.chat_id INTO v_chat_id
  FROM chat_members cm1
  JOIN chat_members cm2 ON cm1.chat_id = cm2.chat_id
  JOIN chats c ON c.id = cm1.chat_id
  WHERE cm1.user_id = v_current_user_id
    AND cm2.user_id = p_other_user_id
    AND c.chat_type = 'dm'
  LIMIT 1;

  -- Found existing chat
  IF v_chat_id IS NOT NULL THEN
    RETURN v_chat_id;
  END IF;

  -- Create new DM chat
  INSERT INTO chats (chat_type, created_by)
  VALUES ('dm', v_current_user_id)
  RETURNING id INTO v_chat_id;

  -- Add both members
  INSERT INTO chat_members (chat_id, user_id, role)
  VALUES
    (v_chat_id, v_current_user_id, 'member'),
    (v_chat_id, p_other_user_id, 'member');

  RETURN v_chat_id;
END;
$$;
```

### 3. Add `getOrCreatePrivateChat` to `ChatsApi`

**File**: `lib/data/api/chat/chats_api.dart`

Add a new method:

```dart
Future<String> getOrCreatePrivateChat(String otherUserId) async {
  try {
    final result = await supabase.rpc(
      'get_or_create_private_chat',
      params: {'p_other_user_id': otherUserId},
    );
    return result as String;
  } catch (e) {
    exceptionHandler.handle(e);
    rethrow;
  }
}
```

### 4. Create `CreatePrivateChatCommand` notifier

**File**: `lib/features/chat_list/notifiers/command/create_private_chat_command.dart` (new)

Following the existing CQRS pattern (`SendMessageCommand`, `DeleteMessagesCommand`):

```dart
import 'package:go_router/go_router.dart';
import 'package:riverpod_annotation/riverpod_annotation.dart';
import 'package:telegram_clone/core/constants/route_names.dart';
import 'package:telegram_clone/data/api/chat/chats_api.dart';

part 'create_private_chat_command.g.dart';

@riverpod
class CreatePrivateChatCommand extends _$CreatePrivateChatCommand {
  @override
  FutureOr<void> build() {}

  Future<String> run({
    required String otherUserId,
    required GoRouter router,
  }) async {
    final link = ref.keepAlive();
    state = const AsyncLoading();

    state = await AsyncValue.guard(() async {
      final chatId = await ref
          .read(chatsApiProvider)
          .getOrCreatePrivateChat(otherUserId);

      router.pushNamed(
        RouteNames.chat,
        pathParameters: {'chatId': chatId},
      );

      return chatId;
    });

    link.close();
    return state.value as String;
  }
}
```
